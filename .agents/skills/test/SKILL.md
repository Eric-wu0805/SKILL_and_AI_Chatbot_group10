# /test — 手動測試清單生成 Skill

## 目標

根據 `docs/PRD.md` 的功能需求和已實作的程式碼，產出一份完整的手動測試清單。

## 前置條件

- 請先讀取 `docs/PRD.md` 了解功能需求。
- 請先讀取 `app.py` 和 `templates/index.html` 了解已實作的功能。

## 執行步驟

1. **閱讀文件**：讀取 PRD 和程式碼，了解所有功能。
2. **設計測試案例**：為每個功能設計測試步驟與預期結果。
3. **輸出測試清單**：在聊天回覆中直接輸出測試清單。

## 輸出格式

請依照以下結構撰寫測試清單：

```markdown
# 手動測試清單

## 測試環境
- 啟動方式：`uvicorn app:app --reload`
- 測試瀏覽器：Chrome / Edge
- 測試網址：http://localhost:8000

## 測試案例

### TC-01：[功能名稱]
- **前置條件**：...
- **測試步驟**：
  1. ...
  2. ...
- **預期結果**：...
- **測試結果**：⬜ 通過 / ⬜ 失敗

（對每個功能重複以上格式）
```

## 必須涵蓋的測試項目

1. **對話狀態管理**：建立新聊天室、切換聊天室、上下文維持
2. **訊息系統**：發送訊息、接收回覆、訊息格式正確（role/content/timestamp）
3. **上傳圖片或文件**：上傳圖片、上傳文件、檔案大小限制
4. **對話歷史管理**：查看歷史對話、切換對話、刪除對話
5. **回答控制**：重新生成回覆、中止回應
6. **記憶機制**：設定偏好、跨對話記憶保持
7. **工具整合**：觸發工具功能、查看工具回傳結果

## 注意事項

- 每個測試案例需有明確的步驟和預期結果。
- 包含正向測試和負向測試（如：空白輸入、超大檔案等）。
- 測試清單應可作為驗收標準使用。
