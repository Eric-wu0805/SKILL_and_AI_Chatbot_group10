# /models — 資料模型設計文件生成 Skill

## 目標

根據 `docs/PRD.md` 和 `docs/ARCHITECTURE.md` 的設計，產出一份詳細的資料模型設計文件，儲存於 `docs/MODELS.md`。

## 前置條件

- 請先讀取 `docs/PRD.md` 和 `docs/ARCHITECTURE.md` 了解需求與架構。

## 執行步驟

1. **閱讀文件**：讀取 PRD 和架構設計文件，理解資料需求。
2. **設計資料模型**：根據功能需求設計資料表結構、欄位與關聯。
3. **撰寫文件**：依照下方模板，產出完整的 `docs/MODELS.md`。

## 輸出格式（docs/MODELS.md）

請依照以下結構撰寫：

```markdown
# 資料模型設計文件

## 1. 資料庫概覽
- 使用的資料庫系統（SQLite）
- ER Diagram（使用 Mermaid 語法）

## 2. 資料表定義

### 2.1 [資料表名稱]

| 欄位名稱 | 資料型態 | 約束條件 | 說明 |
|---------|---------|---------|------|
| id      | INTEGER | PRIMARY KEY, AUTOINCREMENT | 唯一識別碼 |
| ...     | ...     | ...     | ...  |

**索引**：
- ...

**關聯**：
- ...

（對每個資料表重複以上格式）

## 3. 資料關聯圖

使用 Mermaid ER diagram 呈現資料表之間的關聯。

## 4. 資料遷移策略
- 初始化方式（如何建立資料表）
- 種子資料（是否需要預設資料）

## 5. 資料驗證規則
- 各欄位的驗證規則
- 業務邏輯約束
```

## 注意事項

- 所有資料表的設計需能支撐 PRD 中列出的所有功能。
- 使用 SQLite 相容的資料型態。
- 明確標示 PRIMARY KEY、FOREIGN KEY、NOT NULL、UNIQUE 等約束。
- 考慮效能，為常用查詢欄位建立索引。
