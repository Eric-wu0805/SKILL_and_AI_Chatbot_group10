# /architecture — 架構設計文件生成 Skill

## 目標

根據 `docs/PRD.md` 的需求，產出一份系統架構設計文件，儲存於 `docs/ARCHITECTURE.md`。

## 前置條件

- 請先讀取 `docs/PRD.md` 了解專案需求。

## 執行步驟

1. **閱讀 PRD**：讀取 `docs/PRD.md`，理解功能需求與非功能需求。
2. **設計架構**：根據需求決定技術選型、系統元件、資料流。
3. **撰寫文件**：依照下方模板，產出完整的 `docs/ARCHITECTURE.md`。

## 輸出格式（docs/ARCHITECTURE.md）

請依照以下結構撰寫：

```markdown
# 系統架構設計文件

## 1. 架構概覽
- 高層級架構圖（使用 Mermaid 語法）
- 架構風格說明（如：MVC、RESTful API 等）

## 2. 技術選型

| 層級 | 技術 | 說明 |
|------|------|------|
| 前端 | ...  | ...  |
| 後端 | ...  | ...  |
| 資料庫 | ... | ... |
| 其他 | ...  | ...  |

## 3. 系統元件

### 3.1 前端（Frontend）
- 頁面結構
- 主要互動元件
- 與後端的通訊方式

### 3.2 後端（Backend）
- API 路由設計概覽
- 核心模組與責任分工
- 中介層（Middleware）設計

### 3.3 資料庫（Database）
- 資料庫類型與選用原因
- 資料表概覽（詳細設計見 MODELS.md）

## 4. API 設計概覽

| 方法 | 路徑 | 說明 |
|------|------|------|
| GET  | /... | ...  |
| POST | /... | ...  |

## 5. 資料流

使用 Mermaid sequence diagram 描述主要使用情境的資料流。

## 6. 安全性設計
- 認證與授權機制
- 資料保護策略

## 7. 部署架構
- 執行環境
- 啟動方式
```

## 注意事項

- 架構設計需與 PRD 中的功能需求一一對應。
- 使用 Mermaid 語法繪製架構圖與資料流圖。
- 確保技術選型合理且有說明選用原因。
