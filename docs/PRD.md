# 產品需求文件（PRD）

## 1. 專案概述

- **專案名稱**：AI 聊天機器人（AI Chatbot）
- **專案簡介**：本專案旨在開發一個具備完整功能的 AI 聊天機器人 Web 應用程式。使用者可透過瀏覽器與 AI 進行多輪對話，支援多聊天室管理、檔案上傳、對話歷史回顧、回答重新生成等進階功能。系統整合 Google Gemini API 作為 AI 後端，並使用 SQLite 進行資料持久化。

## 2. 目標與範圍

### 專案目標
1. 打造一個可即時互動的 AI 聊天機器人，提供流暢的對話體驗
2. 實現多聊天室（Session）管理，讓使用者可同時維護多個獨立對話
3. 提供進階功能（檔案上傳、記憶機制、工具整合），提升聊天機器人的實用性

### 目標使用者
- 一般使用者：需要 AI 助手協助日常問答、資料查詢
- 開發者：需要測試 AI 對話功能或整合外部工具

### 使用情境
1. **日常對話**：使用者開啟瀏覽器，建立新聊天室，輸入問題並獲得 AI 回覆，可隨時切換至過去的對話繼續討論。
2. **檔案分析**：使用者上傳一張圖片或文件，AI 根據上傳內容進行分析與回覆。
3. **工具查詢**：使用者詢問即時資訊（如天氣），AI 透過工具整合功能呼叫外部 API 取得結果並回覆。

## 3. 功能需求（Functional Requirements）

### 3.1 核心功能

| 功能名稱 | 說明 | 優先級 |
|---------|------|--------|
| 對話狀態管理 | 支援多聊天室（session），維持上下文一致性 | P0 |
| 訊息系統 | 設計 message 結構（role、content、timestamp），支撐完整對話流程 | P0 |
| 對話歷史管理 | 顯示並切換過去的對話紀錄，支援持續使用 | P0 |
| 上傳圖片或文件 | 支援使用者上傳檔案，並作為對話參考內容 | P1 |
| 回答控制 | 提供重新生成（regenerate）、中止回應等控制能力 | P1 |
| 記憶機制 | 儲存使用者偏好或資訊，實現跨對話持續性 | P1 |
| 工具整合（Tool Use） | 串接外部 API 或系統，使聊天機器人具備實際操作能力 | P2 |

### 3.2 功能詳細描述

#### 對話狀態管理
- **使用者操作流程**：使用者可點擊「新對話」建立新的聊天室，每個聊天室有獨立的對話歷史與上下文。
- **系統回應行為**：系統為每個聊天室建立獨立的 session，發送訊息時攜帶完整的對話歷史給 AI。
- **邊界條件**：聊天室數量無上限；切換聊天室時保留各自的對話狀態。

#### 訊息系統
- **使用者操作流程**：使用者在輸入框輸入文字，按下送出按鈕或 Enter 鍵發送。
- **系統回應行為**：系統將訊息以 `{role, content, timestamp}` 格式儲存至資料庫，並將完整對話歷史送至 AI 取得回覆。
- **邊界條件**：空白訊息不可送出；訊息長度無特定限制。

#### 對話歷史管理
- **使用者操作流程**：側邊欄顯示所有歷史對話列表，點擊可切換至該對話。
- **系統回應行為**：載入對應聊天室的所有訊息並顯示。
- **邊界條件**：支援刪除對話；對話列表按最近更新時間排序。

#### 上傳圖片或文件
- **使用者操作流程**：點擊上傳按鈕選擇檔案，檔案預覽後隨訊息一起送出。
- **系統回應行為**：檔案上傳至伺服器，AI 可讀取圖片內容或文件摘要進行回覆。
- **邊界條件**：支援圖片格式（PNG、JPG、GIF、WebP）；檔案大小限制 10MB。

#### 回答控制
- **使用者操作流程**：在 AI 回覆下方提供「重新生成」按鈕；AI 回覆中可按「停止」中止生成。
- **系統回應行為**：重新生成時重新送出最後一次的對話歷史給 AI；中止時停止接收串流回應。
- **邊界條件**：重新生成會替換最後一條 AI 回覆。

#### 記憶機制
- **使用者操作流程**：使用者在對話中提到的偏好或個人資訊，AI 會自動記憶。
- **系統回應行為**：系統將記憶儲存於資料庫，後續對話自動載入。
- **邊界條件**：記憶跨聊天室共用；使用者可手動清除記憶。

#### 工具整合
- **使用者操作流程**：使用者提出需要即時資訊的問題（如天氣查詢、計算等）。
- **系統回應行為**：AI 判斷需使用工具時，呼叫預定義的函式取得結果，再組合成回覆。
- **邊界條件**：工具呼叫失敗時提供友善錯誤訊息。

## 4. 非功能需求（Non-Functional Requirements）

- **效能需求**：AI 回覆使用串流（streaming）方式，首字回應時間 < 2 秒
- **安全性需求**：API Key 不暴露於前端；上傳檔案進行類型與大小驗證
- **可用性需求**：介面響應式設計，支援桌面與行動裝置
- **相容性需求**：支援 Chrome、Edge、Firefox 等主流瀏覽器

## 5. 資料需求

- **Sessions（聊天室）**：ID、標題、建立時間、更新時間
- **Messages（訊息）**：ID、Session ID、角色（user/assistant/system）、內容、時間戳、附件路徑
- **Memories（記憶）**：ID、鍵值、內容、建立時間、更新時間

## 6. 外部依賴

- **Google Gemini API**：AI 對話生成（需要 API Key）
- **Python 3.10+**：後端執行環境
- **SQLite**：內嵌式資料庫，無需額外安裝

## 7. 驗收標準（Acceptance Criteria）

1. ✅ 可成功啟動伺服器並在瀏覽器中開啟聊天介面
2. ✅ 可建立多個聊天室並在之間切換，各自維持獨立對話
3. ✅ 訊息包含 role、content、timestamp 完整結構
4. ✅ 可上傳圖片或文件並獲得 AI 基於內容的回覆
5. ✅ 可重新生成 AI 回覆
6. ✅ 記憶機制可跨對話保持使用者偏好
7. ✅ 工具整合功能可正常運作（如天氣查詢）
